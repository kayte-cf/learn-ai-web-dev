# AI Usage Guide - Day 2

## Today's Focus

Today you're learning your AI-powered editor. Ironically, this means learning **when NOT to use AI**.

## The Manual First Pattern

We'll use this pattern throughout the course:

1. **Do it manually first** - Type it yourself, understand it
2. **Then use AI** - See how AI approaches the same thing
3. **Compare and reflect** - What did AI do differently?
4. **Synthesize** - Develop intuition for when AI helps

## Good AI Uses Today

- "What does this keyboard shortcut do?"
- "Explain what a command palette is"
- "Why would I use the built-in terminal vs. separate terminal?"

## Practice: Manual vs AI

Try this exercise:

1. **Manual**: Write a short paragraph about yourself in `about-me.txt`
2. **AI**: Ask AI to improve your paragraph
3. **Reflect**: What did AI change? Do you agree with the changes?

The goal is to develop judgment about AI suggestions.

## Warning Signs

If you find yourself:
- Asking AI to do everything
- Not understanding AI's suggestions
- Feeling lost without AI

...slow down. Go back to manual. Understanding comes first.

## Remember

Windsurf's AI is a **tool**, not a crutch. The best developers know when to use it and when to think for themselves.
