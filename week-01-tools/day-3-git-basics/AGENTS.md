# AI Usage Guide - Day 3

## Git Commands to Memorize

Today's commands should become muscle memory. Don't rely on AI to tell you these:

```bash
git init
git status
git add <file>
git add .
git commit -m "message"
git log
git log --oneline
```

## Good AI Questions

- "What does a Git commit actually store?"
- "Why is Git called 'distributed' version control?"
- "What's the difference between Git and GitHub?"
- "I got this error: [paste error]. What does it mean?"

## When to NOT Use AI

- Don't ask AI "what command do I use to commit?"
- Don't ask AI to write your commit messages
- Don't skip understanding the staging area concept

## Understanding Over Memorizing

Git has LOTS of commands. You don't need to memorize all of them.

What matters today:
1. **Understand the workflow**: edit → stage → commit
2. **Know these 5 commands** by heart
3. **Everything else** you can look up or ask AI

## Commit Message Practice

Write commit messages yourself. They should:
- Be short (50 characters or less)
- Describe WHAT changed
- Start with a verb (Add, Fix, Update, Remove)

Good: `Add README with project description`
Bad: `changes`
Bad: `asdfasdf`
