# AI Usage Guide - Day 4

## Commands to Know by Heart

After today, these should be muscle memory:

```bash
git clone <url>
git status
git add .
git commit -m "message"
git push
git pull
```

## Good AI Questions Today

- "What's the difference between `git pull` and `git fetch`?"
- "I got a merge conflict - what does that mean?"
- "How do I undo my last commit?"
- "What does 'origin' mean in Git?"

## When to NOT Use AI

- Don't ask AI what commands to run for basic workflow
- Don't ask AI to write your commit messages
- Don't skip understanding local vs remote concept

## Authentication Troubleshooting

If you have trouble pushing, AI can help:
- "I got 'authentication failed' when pushing to GitHub"
- "How do I set up SSH keys for GitHub?"
- "What is a personal access token?"

## The Workflow to Memorize

```
edit → status → add → commit → push
```

Say it out loud. Do it 10 times. Make it automatic.

## Commit Message Guidelines

Your commit messages should tell a story. Someone reading your history should understand what you did:

```
Good history:
- Add Week 1 progress notes
- Add goals for Week 2
- Fix typo in progress notes

Bad history:
- stuff
- more stuff
- asdf
- fixed it
```
