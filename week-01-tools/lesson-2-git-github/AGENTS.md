# AI Usage Guide - Lesson 2

## When to Use AI Today

### Good Uses
- Ask AI to explain Git concepts you don't understand
- Ask AI what an error message means
- Ask AI to help you write a better commit message
- Ask AI to explain what a command does before you run it

### Examples of Good Questions
- "What's the difference between `git add .` and `git add -A`?"
- "I got 'fatal: not a git repository' - what does that mean?"
- "How do I undo my last commit?"
- "What makes a good commit message?"

### Avoid Using AI For
- Don't ask AI to just give you the commands to copy-paste
- Type the commands yourself even if AI shows them
- The goal is to understand the workflow, not just complete the steps

## AI Ethos for This Session

Git commands can feel intimidating at first. It's okay to ask AI to explain what a command does BEFORE you run it. Understanding > speed.

If you're stuck:
1. Read the error message carefully
2. Run `git status` to see what's happening
3. Ask your instructor
4. Then ask AI with the specific error

## Commands to Know By Heart

By end of today, you should be able to type these from memory:
- `git status`
- `git add .`
- `git commit -m "message"`
- `git push`
- `git pull`
- `git clone [url]`

## Good Commit Message Examples

```
# Good
"Add profile introduction to README"
"Complete lesson 2 exercises"
"Fix typo in progress tracker"

# Bad
"update"
"changes"
"asdfasdf"
"fixed stuff"
```

A good commit message answers: "What did this change do?"
